# 33Chat 기능명세서
## 1. 사용자 기능 (User Services)
### 1.1 계정 관리
- **회원가입**: 이메일, 비밀번호, 유저네임, 이름을 입력받아 저장한다.(이메일/유저네임 중복체크)
- **로그인**: Spring Security와 JWT를 사용하여 인증을 처리한다.
- **정보 수정**: 로그인된 사용자는 자신의 유저네임과 비밀번호를 수정할 수 있다.
- **회원 탈퇴**: 탈퇴 시 사용자의 상태를 WITHDRAWN 으로 변경하고, 관련 채팅 세션을 논리 삭제 또는 물리 삭제한다.
- **비밀번호 찾기**: 이메일로 임시 재설정 토큰을 발송하여 비밀번호를 초기화한다.

### 1.2 플랜 및 결제
- **플랜 조회**: 현재 사용 가능한 플랜 목록(FREE, PRO 등)과 혜택을 조회한다.
- **가상 결제 실행**: 사용자가 플랜을 선택하고 결제를 요청하면, 가상결제 로직을 통해 성공 처리를 한다.
- **플랜 변경 적용**: 결제 성공 시 사용자의 plan_id 를 변경하고, 해당 플랜의 limit_tokens 만큼 remaining_tokens 를 즉시 충전한다.

## 2. AI 포털 기능 (AI Services)
### 2.1 채팅 관리
- **세션 생성**: 사용자가 첫 질문을 던지면 새로운 채팅 세션이 생성된다. (첫 질문의 일부가 세션 제목)
- **채팅 내역 조회**: 사용자는 자신이 생성한 모든 채팅 세션 리스트와 특정 세션 내의 메시지 로그를 조회할 수 있다.
- **내역 삭제**: 사용자가 세션을 삭제하면 해당 세션에 포함된 모든 chat_messages 도 함께 삭제된다.

### 2.2 AI 질문 및 답변
- **모델 제한**: 사용자의 플랜에 허용되지 않은 AI 모델로 요청 시 거부 응답을 보낸다.
- **토큰 체크**: 질문 전 remaining_tokens를 확인하여 부족할 경우 "토큰 부족" 메시지를 반환한다.
- **답변 및 차감**: AI 응답이 완료되면 사용된 토큰량을 계산하여 사용자의 remaining_tokens 에서 차감하나고 메시지를 저장한다.

## 3. 관리자 기능 (Admin Services)
### 3.1 사용자 및 플랜 관리
- **계정 제어**: 특정 사용자의 상태를 LOCKED 로 변경하여 로그인을 차단하거나, 강제 탈퇴를 처리한다.
- **권한 부여**: 특정 사용자에게 ADMIN 권한을 부여하거나 회수한다.
- **플랜 정책 설정**: 관리자 페이지에서 각 플랜의 가격, 제공 토큰량, 허용 모델 리스트를 실시간으로 수정한다.

### 3.2 대시보드 및 통계
- **현황집계**:
  - 전체 사용자 수 및 플랜별 가입자 비율 시각화 데이터 제공.
  - AI 모델별 총 사용 토큰량 합계 표시.
- **결제 모니터링**: 일자별 가상 결제 성공 횟수 및 총 매출 합산 데이터 제공.
