# AI 채팅 서비스 요구사항 명세서

## 1. 사용자 (users)
- 사용자는 아래 정보를 유지해야 한다.
  - 사용자 번호(id): USER 접두사 UUID
  - 이메일(email): 로그인 계정 (Unique)
  - 비밀번호(password): 암호화된 텍스트
  - 유저네임(username): 서비스 활동명 (Unique)
  - 역할(role): 권한 등급 (USER, ADMIN)
  - 상태(status): 계정 상태 (ACTIVE, LOCKED, WITHDRAWN)
  - 현재 플랜 번호(plan_id): 현재 구독 중인 플랜 외래키
  - 잔여 토큰량(remaining_tokens): AI 서비스 이용 가능 잔량
  - 플랜 만료 일시(plan_end_date): 유료 플랜 종료 시점
  - 비밀번호 재설정 토큰(reset_token): 비밀번호 찾기용 임시 토큰
  - 토큰 만료 일시(token_expiry): 재설정 토큰의 유효 시간
  - 생성 일시(created_at)
  - 수정 일시(updated_at)
- 사용자는 사용자 번호(id)로 식별한다.
- 사용자는 사이트에 가입하기 위해 이메일(email), 비밀번호(password), 유저네임(username)을 입력해야 한다.
  - 사용자가 사이트에 가입하면 사용자 번호(id)와 초기 역할(role: USER), 기본 플랜(BASIC), 초기 토큰(5,000개)이 부여된다.
  - 이메일과 유저네임은 시스템 내에서 중복될 수 없다.
- 사용자는 본인의 프로필 정보를 수정하거나 회원 탈퇴를 할 수 있다.
  - 탈퇴 시 상태는 WITHDRAWN으로 변경되며, 이후 해당 계정의 서비스 이용은 제한된다.
- 관리자는 사용자 관리 기능을 통해 특정 계정의 상태를 강제로 변경(LOCKED 등)할 수 있다.

## 2. 서비스 플랜 (plans)
- 플랜은 아래 정보를 유지해야 한다.
  - 플랜 번호(id): PLAN 접두사 UUID
  - 플랜명(name): 등급 이름 (Unique)
  - 월간 제공 토큰량(limit_tokens): 해당 등급의 최대 가용 토큰
  - 접근 가능 모델(available_models): 허용된 AI 모델 목록
  - 가격(price): 구독 결제 금액
- 사용자는 보유한 플랜 등급에 따라 서비스 이용 권한을 차등 부여받는다.
  - BASIC: 일반 대화(ASK) 기능만 제공.
  - PRO: 웹 페이지 요약 및 번역(SUMMARY) 기능 추가 제공.
  - PREMIUM: 유튜브 영상 요약(YOUTUBE)을 포함한 모든 기능 제공.
- 관리자는 플랜의 가격이나 제공 토큰량 등 정책 설정을 수정할 수 있다.

## 3. 채팅 세션 및 메시지 (chat_sessions, chat_messages)
- 채팅 세션은 사용자와 AI 간의 대화 맥락을 유지하기 위해 생성된다.
  - 세션 번호(id), 소유 사용자(user_id), 제목(title), 서비스 유형(chat_type) 정보를 포함한다.
- 채팅 메시지는 개별 대화 내역을 저장한다.
  - 메시지 번호(id), 소속 세션(session_id), 발화 역할(role), 내용(content), 사용 모델(model_name), 소모 토큰(used_tokens) 정보를 포함한다.
- 메시지 전송 시 사용자의 잔여 토큰이 소모 토큰보다 적으면 요청이 거부된다.
- 사용자가 채팅 세션을 삭제하면 해당 세션에 포함된 모든 메시지는 함께 삭제(Cascade)된다.

## 4. 결제 (payments)
- 결제는 사용자의 플랜 구독 및 업그레이드를 위해 수행된다.
  - 결제 번호(id), 외부 결제 식별자(payment_id), 결제자(user_id), 구매 플랜(plan_id), 금액(amount), 상태(status) 정보를 관리한다.
- 사용자가 결제에 성공하면 즉시 해당 플랜으로 업그레이드된다.
  - 업그레이드 시 잔여 토큰은 해당 플랜의 최대치로 재설정되며, 유료 플랜은 30일의 만료 기간이 설정된다.
- 외부 PG(포트원)와의 결제 검증을 거친 후 정상적인 건에 대해서만 플랜 반영을 수행한다.

## 5. 시스템 관리 (admin)
- 관리자는 시스템의 자원을 모니터링하고 제어하기 위한 기능을 가진다.
  - 모든 사용자의 목록 조회 및 개별 계정 상태 조정을 수행한다.
  - 전 사용자 토큰 수동 초기화 기능을 통해 모든 사용자의 잔여 토큰을 플랜 한도에 맞춰 갱신할 수 있다.
  - 플랜별 사용자 분포 및 AI 모델별 사용량 통계 데이터를 조회한다.
