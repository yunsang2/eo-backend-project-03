# 2차 프로젝트 2팀 회의록

- 날짜: 2026년 3월 18일
- 참여자: 윤상이 (팀장), 현길용, 김희성

## 오전 회의 내용

- 카카오톡 결제 시스템 도입을 위한 API 명세서 검토 및 백엔드 연동 아키텍처 확정.
- 프론트엔드 메인 UI 중 미완료 상태였던 팝업창(모달) 기능의 디자인 및 이벤트 처리 방식 토의.

## 작업 목표

- 카카오페이 API 결제 연동: 결제 준비(Ready) 및 승인(Approve) 프로세스 구현을 통한 실제 결제 흐름 구축.
- 프론트엔드 팝업 UI 구현: 사용자 알림 및 결제 확인을 위한 모달 창 작성 및 메인 페이지 통합.
- 데이터베이스 연동: 결제 성공 시 주문 상태 변경 및 유저 정보 업데이트 로직 설계.

## 작업 과정

- 카카오페이 개발자 센터를 통해 가맹점 테스트 키를 발급받고, Spring Boot 환경에서 RestTemplate을 활용한 API 통신 로직 구현.
- 결제 요청 시 생성되는 TID(결제 고유 번호)를 세션에 저장하여 승인 단계에서 검증하는 보안 절차 적용.
- 프론트엔드에서 팝업창이 뜰 때 배경 스크롤을 방지하고, 직관적인 닫기 버튼 및 외부 영역 클릭 이벤트를 추가하여 UX 개선.

## 작업 회고

- 백엔드 결제 시스템 작업 시, 단순히 API 응답을 받는 것뿐만 아니라 결제 취소나 승인 실패와 같은 다양한 예외 상황(Exception Handling)에 대한 대응이 필수적임을 확인했음.
- 프론트엔드 팝업 작업에서는 기존 CSS 레이아웃과의 간섭을 해결하며, UI 구성 요소 간의 우선순위 정립이 중요함을 체감함.

## 내일 목표

- 결제 완료 데이터와 연동하여 실시간으로 유저의 잔여 토큰(remainingTokens)이 차감되는 기능 최종 마무리.
- 프론트엔드와 백엔드 간의 전체적인 API 통신 안정성 점검 및 UI 최종 수정.
