# 2차 프로젝트 2팀 회의록

- 날짜: 2026년 3월 17일
- 참여자: 윤상이 (팀장), 현길용, 김희성

## 오전 회의 내용

- 소셜 로그인 구현을 위한 기술 스택(Spring Security OAuth2) 학습 및 기존 JWT 시스템과의 통합 방식 토의.
- 프론트엔드 메인 페이지 구성 중 팝업창을 제외한 나머지 HTML/CSS 요소 작업 진행 상황 공유.

## 작업 목표

- 앨런 AI(Alan AI) API 호출 테스트: 실제 API 통신을 통해 응답 데이터 구조 분석 및 토큰 파싱 로직 설계.
- 실시간 토큰 관리 시스템: AI 답변 수신 직후 유저의 remainingTokens를 즉각 차감하고 DB에 반영하는 기능 구현.
- 프론트엔드 개발: 사용자 편의를 위한 팝업창 제외 잔여 UI 요소 작성 완료.

## 작업 과정

- 구글 클라우드 콘솔 설정을 통해 발급받은 Client ID와 Secret을 활용하여 소셜 로그인 환경 구축 시나리오 수립.
- 일반 로그인과 소셜 로그인의 프로세스 차이(비밀번호 관리 등)를 분석하고, 기존 UserEntity 구조를 재사용하는 방향으로 결정.
- 타 서비스 API 연동 범위에 대해 팀 내 추가 논의 진행 (확장 가능성 검토).

## 작업 회고

- 단순 API 호출 수준을 넘어 소셜 로그인은 기존 회원 체계와의 매핑, 리다이렉트 URI 보안, 정보 수정 시 비밀번호 변경 제한 등 고려할 예외 상황이 많음을 인지함.
- 설계 단계에서 생각했던 것보다 소셜 데이터 매핑 과정이 복잡하여, 보안 취약점 방지를 위한 추가적인 학습과 정교한 로직 구성이 필요함을 체감함.

## 내일 목표

- 구글 소셜 로그인 실제 구현 및 정상 로그인 시 JWT 토큰 발급 테스트 진행.
- 프론트엔드 미완료분(팝업창 등) 기능 구현 및 전체적인 HTML/JS 연동 마무리.
