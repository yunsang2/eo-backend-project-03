- 날짜: 2026년 3월 16일
- 참여자: 윤상이 (팀장), 현길용, 김희성

## 오전 회의 내용

- 소셜 로그인 구현 전 학습 및 방법 토의
- 프론트 팝업 제외한 부분 작성

## 작업 목표

- 앨런 AI API 실제 호출 테스트 및 답변 내 토큰 정보 파싱 로직 구현
- AI 답변 수신 직후 유저의 잔여 토큰을 즉시 차감하는 실시간 반영 기능 개발
- 프론트 팝업을 제외한 남은 부분 작성

## 작업 과정

- 구글 api 설정을 통해 구글 클라우드 에서 발급한 id 와 비번을 가지고 로그인을 구현할 수 있다는걸 알게됨
- 다른 연동 api들을 연동 할지 말지에 대해 의논 필요

## 작업 회고

- 그냥 api 만 받아와 구현하면 될줄 알았지만 의외로 생각해야하는 부분이 많맜으며, 구상한 내용보다 더 복잡해서 조금더 학습이 필요할 것 같다
- 생각할 부분들은 주로 로그인시 소셜로그인은 일반 로그인과 다르기에 정보 수정중 비번 변경및 nickname 변경이 불가해야한다는 점과 소셜 로그인 연동 시, 리다이렉트 URI 설정과 사용자 데이터 매핑 과정에서의 보안 취약점을 방지 해야한다는 점이 있다
