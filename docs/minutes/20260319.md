# 2차 프로젝트 2팀 회의록

- 날짜: 2026년 3월 19일
- 참여자: 윤상이 (팀장), 현길용, 김희성

## 오전 회의 내용

- 가상 결제 시스템 도입을 위한 포트원(Portone) API 연동 방식 및 보안 검증 절차 논의
- 결제 성공 시 사용자의 플랜 정보를 BASIC에서 PRO/PREMIUM으로 자동 승격하는 로직 확정

## 작업 목표

- 가상 결제 연동: 포트원 SDK를 활용한 결제창 호출 및 백엔드 사후 검증 API 구현
- 결제 이슈 해결: V1 연동 시 발생한 '존재하지 않는 결제정보' 에러 원인 파악 및 해결
- 플랜 승급 자동화: 결제 검증 완료 후 DB 내 유저 플랜 업데이트 및 토큰 즉시 충전 기능 구현

## 작업 과정

- V1 트러블슈팅: IamportClient를 이용한 V1 방식의 인증 실패 및 결제 내역 조회 오류(500 에러) 분석 진행
- V2 마이그레이션: V1의 설정 충돌 문제를 해결하기 위해 포트원 V2(신규 API)로 아키텍처를 전격 교체 및 연동 성공
- 백엔드 검증 로직 완성: PaymentService에서 포트원 V2 API를 통해 결제 상태와 금액 위변조 여부를 체크하는 로직 구현 및 테스트 통과

## 작업 회고

- 기술적 유연성: 특정 버전(V1)에서 해결되지 않는 라이브러리 이슈 발생 시, 신속하게 상위 버전(V2)으로 전환하여 문제를 해결하는 의사결정의 중요성을 체감함
- 데이터 정합성: 단순 결제 완료 처리에 그치지 않고, 서버 단에서 실제 결제 금액과 플랜 가격을 2차 검증함으로써 서비스 보안성을 강화함

## 내일 목표

- 매출 통계 시스템: 결제 데이터를 바탕으로 관리자 대시보드 내 실시간 매출 집계 UI 연동 및 검증
- 서비스 최종 점검: 앨런 AI 대화 내역 저장 로직과 결제된 플랜별 권한(토큰 제한 등) 최종 매칭 테스트
